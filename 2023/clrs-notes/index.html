Chapter 5. Probabilistic Analysis and Randomized Algorithms

5.4.4 The on-line hiring problem

  • 注: 此问题描述中索引从 1 开始.
  • 问题: n 个面试者, 依次面试, 面试完立刻决定是否录取. 使用以下策略: 前 k 个不录取, 记录其最高分 X, 从 k+1 个面试者开始, 若分数大于 X 则录取. 求如何确定 k, 使最终恰好录取者分数为最高的概率最大.
  • 结论: 使 k = n / e, 此时能达到的概率最大, 至少为 1 / e.
  • 证明过程: 考察事件概率, 对于确定的 k, 成功录取最高分者的概率 P=∑k<i≤nPi=∑k<i≤nAi⋅BiP = \sum_{k<i \leq n}{P_i} = \sum_{k<i \leq n}{A_i \cdot B_i}P=∑k<i≤n​Pi​=∑k<i≤n​Ai​⋅Bi​.
    • 其中 PiP_iPi​ 表示我们选择了第 i 人且成功的概率.
    • AiA_iAi​ 表示实际最高分者正好为第 i 人的概率
    • BiB_iBi​ 表示 (k, i) 范围内的人没有被我们选择的概率, 等价于 [1, i) 范围中分数最高者位于 [1, k] 内
    • 上述等式成立的原因是: 各 PiP_iPi​ 相互独立, 且对于特定的 i, AiA_iAi​ 和 BiB_iBi​ 独立
    • 通过不等式积分放缩技巧, 可以求出 P 对 k 的关系式, 进一步求其极值即可

Exercise 6.1-7

  • 重要定理: 包含 n 个节点的堆, 其叶节点的索引位于区间 [n / 2, n-1] 内.
  • 证明: 考虑索引为 k 的节点
    • 当 k >= n / 2 时, 假设其存在的左子节点, 则左子节点的索引 leftChildIndex(of: k) >= n + 1, 这是不可能的, 更不用说右子节点了
    • 当 k <= n / 2 - 1 时, 其左子节点索引 leftChildIndex(of: k) <= n - 1, 这一定是一个有效的索引

6.2 Maintaining the heap property

堆的核心操作是若某结点不满足堆要求, 则能快速调整成合法的新堆.

算法说明

  • 前提: 该节点的左右子节点若存在, 则他们本身不许是合法的堆
  • 结果: 调整完后, 以该节点为根节点的子堆是合法的
  • 实现: 若 index 指示的根节点不满足堆的要求, 则将该值向叶节点递归调整.
    • 递归核心: 将根节点和其左右子节点中, 按 comparator 最小的节点进行互换. 然后在被调整的子节点处开始递归.
    • 递归终止条件: 当前节点和其子节点相比已经是最小 (包括不存在子节点)
  • 时间复杂度: O(lg⁡n)O(\lg n)O(lgn)

6.4 The heapsort algorithm

通过使用中间数据结构: 堆, 对数组进行排序.

算法说明:

  • 核心思想: 不断将当前所有元素生成堆, 并把堆顶移除, 就可以产生一个单调序列.
    • 将堆顶和最后的叶节点交换, 并缩小堆的元素范围, 可以直接完成移除堆顶的操作, 同时只需要对根节点进行一次 heapifyNode就可以立刻维持堆的有效性.
  • 注意点:
    • 由于堆顶被移除后, 放到了最后, 因此最终输出的序列是按 comparator 的降序序列, 而非升序序列. 即: 升序序列需要最大堆, 降序序列需要最小堆.
  • 时间复杂度: 上限 O(nlg⁡n)O(n \lg n)O(nlgn).
  • 排序特性: in-place (内排序)

Exercise 6.4-5 对时间复杂度下限的补充

当堆中元素均不相同时, 可以证明此时堆排序时间复杂的的下限也为: Ω(nlg⁡n)\Omega(n \lg n)Ω(nlgn), 配合始终成立的上限, 说明这种条件下, 堆排序的时间复杂度为 Φ(nlg⁡n)\Phi(n \lg n)Φ(nlgn)

  • 参考: Schaffer 和 Sedgewick 1992 年的论文.
  • 证明:
    • 考虑一个 k 层的满堆, 其有 nnn 个结点(n 为奇数). 其中叶节点为 ⌈n/2⌉\lceil n / 2\rceil⌈n/2⌉ 个. 我们只看堆排序流程中产生的前 ⌈n/2⌉\lceil n / 2\rceil⌈n/2⌉ 个值所花费的时间, 这些值是输出结果中最小的那些值(以 comparator 为比较条件).
    • 这最小的 ⌈n/2⌉\lceil n / 2\rceil⌈n/2⌉ 个结点中, 有多少个是位于第 k 层的叶节点呢? 事实上不可能都是叶节点, 精确地说, 不会超过 ⌈n/4⌉\lceil n / 4\rceil⌈n/4⌉ 个.
    • 这是因为根据堆的数学特性, 由 a 个叶节点支撑起的内部节点至少有 a - 1 个. 因此假如上述那些叶节点超过 ⌈n/4⌉\lceil n / 4\rceil⌈n/4⌉ 个, 则他们加上他们撑起的内部节点将多于 ⌈n/2⌉\lceil n / 2\rceil⌈n/2⌉ 个. 而这些节点按照堆的定义, 都是小于或等于排序流程中前 ⌈n/2⌉\lceil n / 2\rceil⌈n/2⌉ 个产生的值. 这在数量上就产生了矛盾.
    • 这说明, 最小的 ⌈n/2⌉\lceil n / 2\rceil⌈n/2⌉ 个结点中, 有至少 ⌈n/4⌉\lceil n / 4\rceil⌈n/4⌉ 个不为叶节点. 根据堆排序的算法, 某个节点如果于一次迭代中被最后于堆顶清除出堆, 它只有 2 种可能. (1) 在迭代开始时作为最后一个节点被 swap 到堆顶, (2) 在 heapifyNode 过程中, 被不断交换到堆顶. 而在前 ⌈n/2⌉\lceil n / 2\rceil⌈n/2⌉ 次迭代中, 只有第 k 层的叶节点可能被 swap, 因此这些至少 ⌈n/4⌉\lceil n / 4\rceil⌈n/4⌉ 个结点只能是从中间不断交换到堆顶的. 我们只需计算这些节点被交换到堆顶需要的时间即可.
    • 在最优情况下, 这 ⌈n/4⌉\lceil n / 4\rceil⌈n/4⌉ 个节点都离堆顶很近, 那总时间也大于 ∑0≤d<lg⁡n4d⋅2d=Ω(nlg⁡n)\sum_{0\leq d < \lg \frac{n}{4}} d\cdot 2^d = \Omega(n \lg n)∑0≤d<lg4n​​d⋅2d=Ω(nlgn).
      ⃞
  • 注意条件中要求的"堆中元素均不相同"是如何起关键作用的: 如果允许堆中元素相同, 那么上述证明中, “产生的前 ⌈n/2⌉\lceil n / 2\rceil⌈n/2⌉ 个值"将不对应”⌈n/2⌉\lceil n / 2\rceil⌈n/2⌉ 个结点", 而是可能对应很多节点, 从而后续数量论证将不再有意义.

值得注意的是 update 方法. 在实际应用场景中, 输入的 index 通常是算法内部数据, 对应用侧是透明的. 因此在实际应用中, 需要将 Element 实体关联到需要操纵它的 index 上. 一般有以下几种设计策略

  1. 要求业务实体支持一个额外字段来存储算法内部所需的标记数据. 算法内部进行 Element 填入、移动等操作时, 写入这个字段即可. 这类设计的缺点是: 在许多语言中是侵入式的, 需要业务改写其实体结构.
  2. 算法将业务传入的 Element 包裹成一个中间类型, 并要求业务使用算法返回的中间类型. 算法内部对中间类型进行操作. 这个中间类型一般拥有 identity, 也就是引用类型, 这样可以保证任何对 index 的改动都作用在同一实体上. 这类设计减轻了侵入性, 但没有完全消除.
  3. 算法另外提供一个方法, 使应用侧可以知道待修改的 Element 位于什么 index, 以便之后调用 update. 算法内部可以选择使用 dictionary 来建立实体和下标的关系, 每次操作实体时, 根据这个 dictionary 对对应下标记录进行同步更新. 这类设计其实也对业务侧有要求, 即 Element 通常应该是引用类型, 或有 “地址” 等 identity 概念, 否则算法无法建立新旧对象的对应关系.

上面这3种设计方案只是举例, 无论如何设计底层需求都是一致的. 这类问题并非指存在于优先级队列, 是算法实用化的普遍问题. 由于不是算法设计核心问题, 今后不再讨论.

Exercise 6.5-9 优先级队列应用: k-way-merge.

  • 问题: 给出 k 个已排序的数列, 进行归并.
  • 使用 Priority Queue 的实现
    • 思想: 使用一个最大尺寸为 k 的堆, 建立优先级队列. 初始时, 从每个数列中取一个加入队列. 之后不断取出队列顶, 取出的元素即已按要求排序. 每次取出元素时, 从来源相同的数列中加入一个新元素并执行堆的调整. 当队列为空时, 排序完成.
    • 简单证明: 使用以下 invariant 即可.
      • 每次从队列中移除时, 不在队列中的元素一定有另一个元素比他更小, 因此此时的全局最小值一定是堆顶.
      • 当每次移除完堆顶, 除了来源相同的数列中的最前者, 其他元素(无论是来源相同的数列还是不同的数列), 都有另一个元素比他更大.
    • 时间复杂度: O(lg⁡k⋅N)O(\lg k \cdot N)O(lgk⋅N), 其中 NNN 为总元素数.

Problem 6-3 Young Tableaus

  • 问题定义: 一个 n×mn \times mn×m 的矩阵, 每行从左往右, 每列从上倒下均有序排列. 部分位置可以填特殊值+∞+\infty+∞, 同时表示该处为空. 从某种角度看, “Young Tableaus” 是堆的变体, 区别在于有一部分结点在某个根的左右子堆中共享.

[1234623468457910] \begin{bmatrix} 1 & 2 & 3 & 4 & 6 \\ 2 & 3 & 4 & 6 & 8 \\ 4 & 5 & 7 & 9 & 10 \end{bmatrix} ⎣⎢⎡​124​235​347​469​6810​⎦⎥⎤​

  • 算法 dropMin 移除最小的元素, 同时保持矩阵合法.
    • 类似 heapify, 在左上角填入 +∞+\infty+∞, 并不断调整. 每次调整的依据为: 当 M(i, j) 位置的元素不合法时, 选择 M(i + 1, j) 和 M(i, j + 1) 中的小者与之交换.
    • 时间复杂度: O(m+n)O(m + n)O(m+n).
      • 也可以由递推式 T(p)=T(p−1)+O(1)T(p) = T(p - 1) + O(1)T(p)=T(p−1)+O(1) 得到, 其中 p=m+np = m + np=m+n
    • 正确性证明: 使用以下 invariant: 调整前, 只有第 i 行和第 j 列可能不满足条件, 调整后, 只有第 i + 1 行和第 j + 1 行可能不满足条件. 且不满足条件的行和列中, 去掉这个 +∞+\infty+∞ 以外的部分仍然合法.
  • 算法 insert 插入一个新的值.
    • 类似 insert/increase, 在右下角填入值, 并不断调整. 每次调整的依据为: 当 M(i, j) 位置的元素不合法时, 选择 M(i - 1, j) 和 M(i, j - 1) 中的大者与之交换.
    • 时间复杂度: O(m+n)O(m + n)O(m+n).
    • 正确性证明: 使用以下 invariant: 调整前, 只有第 i 行和第 j 列可能不满足条件, 调整后, 只有第 i - 1 行和第 j - 1 行可能不满足条件. 且不满足条件的行和列中, 去掉这个 +∞+\infty+∞ 以外的部分仍然合法.
  • 不断使用 insert 和 dropMin, 显然可以使用 n×nn \times nn×n 的 Young Tableaus 在 O(n3)O(n^3)O(n3) 内完成整体排序
  • 2维查询, 查找元素 x 是否位于某 Young Tableaus 中.
    • 使用递归思想. 对于矩阵 M[i...i'][j...j'] 考察 M[i, j'] 和 M[i', j] 之间的关系.
      • 如果 M[i, j'] <= M[i', j], 则可知第一行与最后一行不相交, 根据 x 和它们的关系, 可以缩小查找范围.
        • 若 x > M[i, j'] 则去掉第 i 行
        • 若 x < M[i', j] 则去掉第 i' 行
      • 如果 M[i, j'] > M[i', j], 则可知第一列与最后一列不相交, 根据 x 和它们的关系, 可以缩小查找范围.
        • 若 x > M[i', j] 则去掉第 j 列
        • 若 x < M[i, j'] 则去掉第 j' 列
      • 每次迭代均确保可以去掉 1 列或 1 行. 当只剩 1 行或 1 列时, 进行线性查询即可. 实则可由2维查询的逻辑退化得到.
    • 一个更简单的算法: 从左上角或右上角开始. 例如从右上角开始:
      • 如果当前 x < M[i, j'] 则去掉最第 j' 列
      • 如果当前 x > M[i, j'] 则去掉最第 i 行
    • 时间复杂度: 由 T(m+n)=T(m+n−1)+O(1)T(m + n) = T(m + n - 1) + O(1)T(m+n)=T(m+n−1)+O(1) 解得: T(m+n)=O(m+n)T(m + n) = O(m + n)T(m+n)=O(m+n).
